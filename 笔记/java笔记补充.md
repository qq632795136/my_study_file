# 抽象类与方法

1. 抽象类中不一定包含抽象方法，但是抽象方法所在的类必须是抽象类
2. 不能直接new抽象类对象，只能通过子类继承并覆盖重写所有抽象方法，抽象类的子类必须重写所有抽象方法，否则编译报错，除非子类也是抽象类
3. 抽象方法可以有构造方法，但是也不能直接使用，只能在子类中使用.super方法使用，而且在子类创建实例对象时，父类的抽象构造方法先运行，然后再是子类的构造方法（如果有的话）
4. 创建子类并使用

# 多线程

线程开启不一定立即执行，而是由我们的CPU调度。

1.继承Thread类

具备多线程能力

启动线程：子类对象.start()

不建议使用：避免OPP单继承局限性

2.实现Runnable接口

具有多线程能力

传入对象+Thread对象.start()

推荐使用：避免单继承局限性，灵活方便，方便同一个对象被多个线程使用

  ## 线程不安全

多个线程操作同一个资源的情况下，线程会不安全，数据出现紊乱
